# 门面\(Facades\)

### 知识点整理

```
1.简介
    门面为应用服务容器中的绑定类提供了一个“静态”接口.(服务容器中底层类的静态代理)
    所有门面都定义在Illuminate\Support\Facades目录下
2.何时使用门面
使用过多门面会让类变得越来越大,当然没有依赖注入的构造函数来的直观.
    门面vs.依赖注入
        门面使用了动态方法对服务容器中解析出来的对象方法调用进行了代理.
    门面vs.辅助函数
        内置了许多辅助函数用于执行通用任务.
        比如生成视图,触发事件,分配任务,以及发送HTTP响应等.很多辅助函数提供了和相应门面一样的功能.
        门面和辅助函数之间并不存在实质性差别.
3.门面工作原理
    门面就是一个为容器中对象提供访问方式的类.
    门面都继承Illuminate\Support\Facades\Facade基类.
    门面类只需要实现一个方法,getFacadeAccessor.
    正是 getFacadeAccessor 方法定义了从容器中解析什么,
    然后 Facade 基类使用魔术方法 __callStatic() 从你的门面中调用解析对象.
```
